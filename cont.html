2.4 Control de Flujo

El control de flujo es un mecanismo esencial en los sistemas de telecomunicaciones y redes de datos, cuyo propósito es regular la cantidad de información que el emisor puede enviar al receptor, evitando que este último se vea saturado. Su objetivo principal es asegurar que la transmisión de datos sea ordenada, eficiente y confiable.

En una comunicación sin control de flujo, el receptor podría recibir más información de la que puede procesar, provocando pérdida de datos, errores y retransmisiones innecesarias.

Mecanismo de Ventana Deslizante

Uno de los métodos más utilizados para implementar el control de flujo es el mecanismo de ventana deslizante. Este método permite que el emisor envíe varias tramas antes de recibir una confirmación por parte del receptor, aumentando el rendimiento del sistema de comunicación.

Características principales

  • El emisor puede enviar varias tramas antes de recibir una confirmación (ACK).
  • La ventana deslizante tiene un tamaño fijo de “n” celdas numeradas (1…n).
  • Se pueden enviar hasta n tramas antes de recibir una confirmación.
  • Un ACK contiene el número de la última trama recibida correctamente.

Cuando el receptor confirma la recepción correcta de las tramas, la ventana se desplaza, permitiendo el envío de nuevas tramas. Este proceso optimiza el uso del canal de transmisión y reduce los tiempos de espera.

Control de Flujo por Software

El control de flujo por software se basa en el intercambio de información adicional entre el emisor y el receptor para indicar el estado del proceso de recepción. Este método utiliza caracteres de control o campos especiales dentro de las tramas de datos.

Mediante este mecanismo, el receptor puede indicar si está listo para recibir más información o si necesita que el emisor detenga temporalmente la transmisión. De esta forma, se evita la saturación del receptor y se mantiene la integridad de los datos.

Ventajas del control de flujo

  • Evita la pérdida de datos.
  • Optimiza el uso del canal de comunicación.
  • Reduce errores y retransmisiones.
  • Permite una comunicación más estable y eficiente.
