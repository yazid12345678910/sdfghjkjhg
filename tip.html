2.4.1 Tipos: Asentimiento y Ventanas Deslizantes

Asentimiento (Parada y Espera)

El asentimiento, también conocido como protocolo de parada y espera, es uno de los métodos más simples para el control de flujo. Su objetivo principal es evitar la saturación del receptor.

En este método, el emisor envía una sola trama y se detiene hasta recibir un acuse de recibo (ACK) por parte del receptor. Solo después de recibir dicha confirmación, el emisor puede enviar la siguiente trama.

Este procedimiento es adecuado cuando se transmiten pocas tramas de gran tamaño. Sin embargo, resulta poco eficiente cuando se transmiten tramas cortas, ya que el tiempo de espera reduce el rendimiento del sistema.

Eficiencia del Asentimiento

La eficiencia se define como la proporción entre el tiempo utilizado para transmitir información útil y el tiempo total del proceso. Se calcula considerando el tamaño de la trama y la velocidad de transmisión del emisor.

  • Sencillo de implementar.
  • Alta confiabilidad.
  • Bajo aprovechamiento del canal.

Ventanas Deslizantes

El mecanismo de ventana deslizante es un método más eficiente y ampliamente utilizado en redes de comunicación. Permite que el emisor envíe varias tramas sin esperar inmediatamente una confirmación.

La ventana define cuántas tramas pueden estar pendientes de confirmación al mismo tiempo. Su tamaño depende de la capacidad del buffer del receptor y del número de secuencia utilizado para numerar las tramas.

Por ejemplo, si se utilizan 3 bits para numerar tramas (módulo 8), el tamaño máximo de la ventana será de 7 tramas (2k − 1). Con 7 bits de numeración, se podrían enviar hasta 127 tramas sin recibir un ACK.

Ventajas de la Ventana Deslizante

  • Mayor eficiencia en el uso del canal.
  • Reduce el tiempo de espera.
  • Mejor rendimiento en enlaces de alta velocidad.

Control de Flujo por Hardware

El control por hardware utiliza líneas físicas adicionales para gestionar el flujo de datos. Un ejemplo común es la interfaz RS-232-C, que emplea señales eléctricas para indicar si el receptor está listo para recibir información.

Estas líneas funcionan como un semáforo: permiten o detienen la transmisión dependiendo del estado del dispositivo receptor. En comunicaciones bidireccionales se requieren al menos dos líneas de control.

  • Alta confiabilidad.
  • Dependiente del medio físico.
  • Requiere cables adicionales.

Control de Flujo por Software

El control por software se basa en el envío de caracteres de control o información incluida en las tramas para indicar el estado del receptor.

Este método ofrece mayor flexibilidad e independencia del medio físico. Un ejemplo común es el uso de protocolos como XON/XOFF o el envío de confirmaciones (ACK) antes de continuar con la transmisión.

  • Mayor versatilidad.
  • No depende de líneas físicas adicionales.
  • Más adecuado para redes modernas.
